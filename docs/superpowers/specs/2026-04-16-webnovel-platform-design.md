# Webnovel & Light Novel Reading Platform — Design Spec

**Date:** 2026-04-16  
**Status:** Approved by user

---

## Overview

A multi-novel reading platform for publishing fan translations of webnovels and light novels. The site hosts translated chapters (currently the "Slime Reincarnation" series, 103+ chapters) and is designed to grow into a full library of series over time.

The defining goal is a high-quality, aesthetically pleasing reading experience — not a generic fan-translation aggregator. Think literary magazine meets modern reading app.

---

## Tech Stack

| Layer | Choice | Reason |
|-|-|-|
| Framework | Next.js (App Router, static export) | Auto-generates pages from Markdown files at build time; scales cleanly as novels grow |
| Content | Markdown (`.md`) files with frontmatter | Rich formatting support (bold, italic, scene breaks, etc.); generated by translation pipeline |
| Deployment | Vercel | Auto-deploys on every git push; free tier covers the use case |
| Fonts | Playfair Display (headings) + Lora (body) + Inter (UI) | Literary feel with excellent readability |
| Styling | Tailwind CSS | Utility-first; fast to build, easy to maintain |

### Git Setup Required
The `F:\TL Website` directory is **not currently a git repository**. Before deployment:
1. Run `git init` in `F:\TL Website`
2. Create a GitHub repo and push
3. Connect the GitHub repo to Vercel for automatic deployments

```

  content/
    novels/
      slime-reincarnation/
        novel.md              ← novel metadata
        chapters/
          ch-001.md
          ch-002.md
          ...
          ch-103.md
      [future-novel-slug]/
        novel.md
        chapters/
  public/
    covers/
      slime-reincarnation.jpg   ← cover art images
```

### Novel Metadata (`novel.md` frontmatter)
```yaml
---
title: "That Time I Got Reincarnated as a Slime"
slug: slime-reincarnation
cover: /covers/slime-reincarnation.jpg
status: ongoing
genres: [Fantasy, Isekai, Adventure]
description: "A salaryman reborn as a slime discovers that even the humblest creature can reshape the world."
---
```

### Chapter Metadata (`ch-103.md` frontmatter)
```yaml
---
title: "Divine Skill — Love is a Ray of Light"
chapter: 103
novel: slime-reincarnation
published: 2026-04-16
---
```

### Translation Pipeline Integration
The existing translation agents (in `Slime/.claude/agents/`) currently output `.txt` files. They will be updated to output `.md` files with the frontmatter above, placed directly into `content/novels/[slug]/chapters/`. A git commit + push then triggers an automatic Vercel redeploy — zero manual steps.

> **Dependency:** Updating the translation agents to output Markdown is a separate task from building the website. Until it is done, chapters can be converted manually from `.txt` to `.md` with the correct frontmatter.

### Cover Art
Novel cover images are referenced in `novel.md` frontmatter but do not currently exist. For v1, either:
- Use a generated placeholder cover (styled CSS gradient matching the novel's colour scheme), or
- Source or commission actual cover art and place it in `public/covers/[slug].jpg`

The site will fall back gracefully to a placeholder if no image file is found.

## Page Designs

### Homepage (`/`)

**Style:** Modern & Clean — white background (`#ffffff`), red accent (`#c0392b`), Playfair Display headings.

**Layout:**
1. **Navbar** — Logo left, nav links (Browse, Genres, Updates), search bar right
2. **Hero banner** — Full-width gradient panel featuring the latest/highlighted novel. Title, description, "Start Reading" + "Continue" CTAs. Novel cover art floated right.
3. **Genre filter pills** — All / Fantasy / Isekai / Romance / Action / Mystery (client-side filter, no page reload)
4. **Novel grid** — 5-column card grid. Each card: cover thumbnail (2:3 ratio), genre tag, title, chapter count, reading progress bar (filled based on localStorage)
5. **Footer** — Logo, tagline, links

### Novel Detail Page (`/novels/[slug]`)

**Layout:**
1. **Cover + metadata** — Large cover art left, title / genres / status / description right
2. **Chapter list** — Numbered list with chapter titles and publish dates, sorted newest-first with toggle to flip order
3. **"Start Reading" / "Continue Reading"** CTA — reads from localStorage to know user's last chapter

### Chapter Reader (`/novels/[slug]/[chapter]`)

**Style:** Light Literary — warm cream background (`#faf7f2`), Lora body font, centred text column (max 680px wide).

**Layout:**
1. **Top bar** — Back link (← Novel title), chapter dropdown selector (jump to any chapter), font controls (Aa), dark/light toggle
2. **Reading progress bar** — Thin red line across full width showing scroll position
3. **Content column** — Centred, 680px max-width, generous padding
   - Novel name (small red label)
   - Chapter title (Playfair Display, 28px)
   - Publish date
   - Chapter body text (Lora, 17.5px, 1.9 line height)
4. **Chapter navigation** — Prev / Next buttons at the bottom
5. **Collapsible sidebar drawer** — Triggered by ☰ icon in top bar. Slides in from left as an overlay (does not push content). Contains: novel cover, novel title, full chapter list with active chapter highlighted. Closes on backdrop click or Escape key.

**Dark mode:** Toggled via button in top bar. Switches to `#0f0d0b` background, `#c0b090` text, `#c9a96e` accent. Preference saved to localStorage.

## Features

### Reading Progress Tracker
- Stored in `localStorage` keyed by novel slug
- Tracks last-read chapter number
- Homepage novel cards show a progress bar (e.g. "65/103 chapters")
- Novel detail page shows "Continue Reading → Chapter 103" CTA

### Dark / Light Mode Toggle
- Button in chapter reader top bar
- Also available site-wide via navbar icon
- Preference persisted in localStorage
- System default (`prefers-color-scheme`) used on first visit

### Font Size & Line Spacing Controls
- Three presets: Compact / Default / Relaxed
- Affects font size (16 / 17.5 / 19px) and line height (1.7 / 1.9 / 2.1)
- Preference saved to localStorage per user

### Search
- Client-side search across novel titles and chapter titles (no server needed)
- Powered by a pre-built search index generated at build time
- Search bar in navbar; results dropdown shows matching novels and chapters

### Genre Tags & Filtering
- Defined in each novel's `novel.md` frontmatter
- Homepage filter pills filter the novel grid client-side
- `/novels` page supports URL-based filtering (`/novels?genre=fantasy`)
